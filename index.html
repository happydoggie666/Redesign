RESPONSIVE REDESIGN

Looking back, it is really bothersome to open a webpage which works fine on PC but is not user-friendly on phones or ipad. Imagine when you are really hungry and want to order food on a phone but the shop doesn’t really have an app. You click on the ordering link and find that everything is floating around. You spend a long time trying to figure out how to use it and where the order button is. Finally you successfully place an order but are also very exhausted.

To avoid such a phenomenon, from the user's perspective, the designer needs to create a product that will display and function properly (at least) most of the time - regardless of the size of the display and the type of device.

Therefore, this project aims to design and build a responsive and user-friendly website in either usability, responsiveness or aesthetics aspects based on low-fidelity and high-fidelity prototypes of an existing interface.

Part 1: Identifying Usability Problem

1.1 Picking a Web Page

Although being ugly is one of its special features and making the website ‘attractive’, the overly crowded text, the overly colorful layout and the overused animation is very overwhelming for both new users and old users.

1.2 Finding Problems

1.2.1 Mental v.s. Conceptual model

Mental Model:

When entering lingsCars trying to rent a car, people would mostly think of:

  1. Look at the menu
  2. Choose the car brand they like or directly click on the specific car they are interested in to see the detail
  3. In the detail session, they would like to see the price and the related contract
  4. Check the review & the information contact of LingsCars
Conceptual model:

The conceptual model that this website provides is:

  1. Provide a large banner which leads to the car list
  2. Show the menu
  3. Provide the car brand
  4. Show the contact information

1.2.2 Usability & Efficiency

  1. Low image-font color contrast
  2. Hard for both new users and old users to spot important information and quickly perform the operation they want
  3. Many photos have the same alt text, which cannot provide much information if the image is not showing
  4. The large empty white region in the bottom

1.2.3 Memorability

Although the website provides many affordances with the usage of icons, it is still hard to recall how to do a single operation. Say, the user wants to rent a car, after they click the “order now” icon, it is hard to tell what should be done next.

1.2.4 Learnability

Although the website provides many affordances with the usage of icons, it is still hard to learn for new users. The overly colorful page and overly crowded icons are just so distracting.

1.3 Accessibility

WebAIM WAVE Report

Accessibility findings:

The website’s main problem is it has many (46) contrast errors and the heading hierarchy/structure is very unclear. There are many redundant and small texts which are not easily recognizable. WAVE also finds that a nearby image contained in the website has the same alternative text, which cannot provide hints for the users of what the image is about if the image is not showing.

My personal opinion:

I agree with the WAVE’s opinion that the website has many contrast errors, the overly colorful page makes it hard for users to find something they are looking for efficiently. Although there are some icons which support affordance, it is still not that distinguishable. The page hierarchy is not clear enough. Many description texts are redundant. Also, the website seems to use too many LING’s portraits and memes.

Annotations:

Low-fidelity
  1. Card Brand Logo Display and White Blank at the bottom issue

    To solve the "card brand logo display" not distinguishable problem, I use an image slider and put the car brand logo alphabetically so that people can scroll and search for the brand they like.

  2. Hard to go back to the menu and jump to other page issue

    In order to solve this problem, I designed a fixed top nav bar in my low-fidelity design. Although I twist the way I do it in my high-fidelity and the final implementation, I still find it useful to solve the issue when users want to go back to the menu and have to scroll up for quite a while to do so. It will improve the efficiency and the user experience.

  3. Too many information issue

    I add a filter function in my low-fidelity design. Although I didn’t implement it in my final version, it helps to filter out the redundant information.

High-fidelity
  1. Card Brand Logo Display and White Blank at the bottom issue:

    To solve the "card brand logo display" not distinguishable problem, I use an image slider and put the car brand logo alphabetically so that people can scroll and search for the brand they like.

  2. Gallery display:

    I adjust the cars gallery’s size and put each car in a flashcard. I add a hover animation to the car flashcard so that if people’s mouse moves to some specific cars, the car name would turn green and the car picture size would be **scaled to 0.9** of its original size. This makes it more clear where people are looking so that people won’t get lost and confused easily.

  3. Quick returning to the top:

    In order for people to quickly return to the top, I add a “TOP button” which floats at the left-bottom of the screen and it will only show up when people scroll down more than 20px from the top.

    I decided to not do a fixed top nav bar and alternately choose this way, as I feel like this fits my design style better. Notice that this differs from my low-fidelity version.

  4. Responsiveness:

    I use media queries + grid + flex box for many elements of my website(i.e. Search bar). To be specific, a media query judges whether a screen’s max-width is below 600px and the website will render differently based on this query.

  5. Responsiveness-2 (flex-box):

    To be specific, the key CSS attributes I use for each session related to flex-box and flex-direction is:

    • Brand Session: "display:flex", "flex-direction:row"
    • Cars / Vans Session: "display:flex", "flex-direction:row"
  6. Responsiveness-2 (padding):

    To be specific, the key CSS attributes I use for each session related to padding is:

    • Brand Session: each logo flashcard: "padding: 0 10vw;"
    • Cars / Vans Session: each product card: "padding: .5rem 0"
  7. Responsiveness-2 (grid):

    To be specific, the key CSS attributes I use for each session related to grid is:

    • Cars / Vans Session: each product card: 
                                      .product_card  {
                                display:grid;
                                grid-template-columns:repeat(auto-fit, minmax(30rem, 1fr));
                                gap:1rem; 
                                padding: 0 10vw;
                                }

      • I also add box shadow "box-shadow: 0 .5rem 1rem rgba(0,0,0,.1);" for the above session mentioned to make it more distinguishable.

  8. Responsiveness-2 (hover):

    I added hovers to multiple elements. I will just list an example besides scaling the image mentioned above.

    When you move the mouse to a car brand logo or a specific car, a red button with white reminder shows up. I implement it through the below CSS

✨Outcome & Reflection✨

As I get more feedback and revisit the results in light of what I have learned, I think there are a few points worth discussing.

For the project display page, I received the following suggestions:

  1. The jumping hover effect is a little bit distracting where things keep on jumping up at you unexpectedly.
  2. It would be nice to have the annotations on the wireframes (low-fidelity prototype and high-fidelity prototype) itself. The annotations should be pointing to the exact area on the website to make it easier to understand how I built my redesigned website.
  3. The image in the high-fidelity design session loads very slowly. I could consider compressing some of them and rearrange the layout.

For the responsive design page itself, I received the following suggestions:

  1. Only having 2 cars per row in the Cars / Vans sections is too few where the images are too large and will cause people to scroll a lot when a lot of cars are present.

  2. The website didn’t really become responsive in the iphone viewpoint. Some texts were cut off. The redesigned page for the mobile layout was hard to navigate as the car brand images and car images were very large.

    To adjust this, I would re-define the media query constraint to make it more adjustable to the iphone setting. Or maybe I will remake a react page for this one instead of an html+css page.

  3. I could add a hamburger menu to make my website more responsive for smaller screen sizes.
